# Lock Room API

REST API for the Lock Room project — an encrypted credentials vault manager.

## Stack

| Layer | Technology |
|---|---|
| Runtime | Bun |
| Framework | Elysia |
| ORM | Drizzle ORM + drizzle-kit |
| Database driver | pg (Pool) |
| Database | PostgreSQL |
| Language | TypeScript (strict) |
| Docs | OpenAPI via `@elysiajs/swagger` (Scalar UI) |

## Prerequisites

- [Bun](https://bun.sh) >= 1.x
- PostgreSQL running (or via Docker Compose at the project root)

## Setup

```bash
cp .env.example .env
# fill in your environment variables
```

Required variables:

```env
DATABASE_URL=
JWT_SECRET=
SERVER_PORT=
CRYPTO_SECRET=
```

## Running locally

```bash
bun install
bun run dev
```

The API starts at `http://localhost:<SERVER_PORT>`.

Interactive docs available at `http://localhost:<SERVER_PORT>/swagger`.

## Scripts

| Command | Description |
|---|---|
| `bun run dev` | Start with hot reload |
| `bun run lint` | Lint with ESLint |
| `bun run format` | Format with Prettier |
| `bun run format:check` | Check formatting |
| `bun run db:generate` | Generate migration with drizzle-kit |
| `bun run db:migrate` | Apply migrations |
| `bun run db:push` | Push schema directly to the database (dev) |

## Project structure

```
src/
  configs/              # env vars centralized by domain
    cors.config.ts
    crypto.config.ts
    database.config.ts
    jwt.config.ts
    logger.config.ts
    server.config.ts
  database/
    connection.ts       # pg Pool
    index.ts            # drizzle instance
    schema/             # drizzle schemas
    migrations/         # generated by drizzle-kit
  exceptions/           # typed exceptions (e.g. NotFoundException)
  plugins/              # Elysia plugins organized by category
    auth/
      jwt/              # JWT plugin
    core/
      error-handler/    # standardized onError handler
    crypto/
      server-crypto/    # encryption plugin
    docs/
      swagger/          # Scalar UI
    infra/
      cors/
      logger/
      rate-limit/
      xss/
  modules/              # application features
    auth/               # login, logout, register
    recovery/           # account recovery
    vault/              # vaults and credentials
    modules.routes.ts   # aggregates all routes
  index.ts              # registers plugins and imports modules.routes.ts
```

## Conventions

- Each module lives in `src/modules/<feature>/` with files named `<feature>.<type>.ts`
- Valid types: `routes`, `service`, `repository`, `schema`, `types`, `constants`, `docs`
- Database queries **only** in `repository` — `service` never accesses `db` directly
- `process.env` is never accessed directly — always through `src/configs/<domain>.config.ts`
- Errors are thrown via typed exceptions in `src/exceptions/` — no bare `new Error()`
- Plugins never use `index.ts` — files are named with explicit suffixes (`.plugin.ts`, etc.)

## Docker

To run via Docker Compose (from the project root):
